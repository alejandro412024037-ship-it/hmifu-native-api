# HMIFU Native API

## Commit Message Format

Always use **Conventional Commits** in **English**:

```
<type>(<optional scope>): <short description>
```

### Types
- `feat` — new feature
- `fix` — bug fix
- `refactor` — code change that neither fixes a bug nor adds a feature
- `docs` — documentation only
- `test` — adding or updating tests
- `chore` — build process, dependency updates, config

### Rules
- Subject line: imperative mood, lowercase, no period, max 72 chars
- Example: `feat(auth): add JWT token refresh endpoint`
- Example: `fix(event): return 404 when event not found`
- Example: `chore: update database config`
